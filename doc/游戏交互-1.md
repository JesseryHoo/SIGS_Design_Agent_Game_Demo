# 游戏交互-1

## 一、项目定位与核心叙事框架
项目名称方向（可选）

「未来校园共创计划」 — 强调参与感与共同创造
「我的校园2050」 — 时间感 + 个人视角，易产生代入感
「校园漫游者」 — 游戏感强，适合非专业公众

核心叙事逻辑
要让非专业公众愿意参与设计，需要解决一个心理障碍："我不懂设计，我的想法有用吗？"
因此整套文案的核心叙事应该是：

"你不需要懂设计，你只需要告诉我们你的感受。每一个在这里生活过的人，都是这座校园最好的设计师。"

这句话作为整个项目的贯穿主线，在每个交互节点上持续强化。

## 二、用户旅程与交互流程设计
整个体验分为五个阶段，每个阶段有对应的交互设计和文案策略。
### 阶段一：进入与吸引（Landing）
目标： 30秒内让用户理解"这是什么"并愿意继续
页面元素：

动态标题动画：校园三维模型缓慢旋转，若干光点在建筑上闪烁，暗示"这里有故事"
主标题（大字）：这座校园，还可以更好
副标题：告诉AI你的想象，它来帮你把它造出来
引导按钮：开始探索 →
底部小字说明：无需注册 · 无需设计经验 · 3分钟完成

关键文案原则： 去除所有专业词汇（不出现"参数化""公众参与""Agent"等术语），用日常语言建立信任。

### 阶段二：场景漫游与选点（Explore）
目标： 让用户在三维校园中自由游览，找到"让自己有感觉的地方"
交互设计：

提供漫游模式（自动飞行巡游）和自由探索模式（鼠标/键盘控制视角）两种入口
漫游路径经过精心设计，覆盖校园典型场景：入口广场、图书馆前庭、宿舍区庭院、架空层、屋顶等
建筑可高亮悬停，显示简短的场景标签，如："午后最热的角落" / "下雨天没有遮挡的路" / "没人去的好地方"——用情绪化语言而非功能性描述来标注空间

引导文案：

漫游开始前：走进这座校园，找一个让你有感觉的地方——可以是你觉得不够好的，也可以是你想象中它本来的样子
悬停提示：点击这里，告诉我们你的想法
等待中的空屏引导：还没找到？试试去图书馆后面看看……


### 阶段三：输入创意（Create）
目标： 让用户用自然语言输入非专业的设计想法，消除表达障碍
交互设计：
输入界面要做到极度低门槛，分三层递进：
#### 第一层：情绪选择（选填）
用图标+词语让用户先描述"感受"，不是功能：
😤 不够用 / 😌 太单调 / 🌿 想要绿色 / ☀️ 采光不好 / 🧍 太拥挤 / 🔇 太吵了
#### 第二层：自由输入框

占位文字（placeholder）示例要具体、生活化，而不是空白或者"请输入描述"：

"这里的长椅太少了，下课想坐下来发发呆"
"希望这个角落能种一棵树，夏天可以乘凉"
"感觉这条路走起来不安全，晚上没有灯"


输入框上方提示语：用你最自然的语言说就好，哪怕只有一句话

#### 第三层：可选补充（折叠展开）

时间偏好：我希望在 [早晨/午后/傍晚/夜晚] 使用这里
使用方式：我想在这里 [独处/和朋友/工作/休息/路过]

提交按钮文案： 让AI来理解我的想法 →（而不是"提交"或"生成"）

#### 阶段四：生成与呈现（Generate）
目标： 让等待过程有参与感，让结果有惊喜感和认同感
等待动效文案（逐步出现，配合加载动画）：

AI正在理解你的想法……
正在分析这个空间的光线与尺度……
正在把你的感受转化为空间语言……
你的校园，正在生长中……

这组文案的目的是让用户感知到"我的语言被认真对待了"，而不是冷冰冰的进度条。
结果呈现页：

生成模型以动态展示方式出现（渐显 + 轻微旋转），营造"揭晓"感
模型旁边附上一段AI的回应文字，用第一人称向用户解释它"理解"了什么：

"你提到希望这里能有更多遮阴和停留的地方。我保留了原有的开放视野，在东侧加入了一组轻质廊架结构，搭配低矮的种植箱，让这个角落在午后变成可以安静坐下来的地方。"

这段文字至关重要——它把技术生成结果转化为一次对话的回应，让用户感到被理解。
底部操作：重新调整想法 / 分享这个设计 / 继续探索其他地方


#### 阶段五：社区沉淀（Gallery）
目标： 展示所有参与者的创意，形成集体共创感
页面概念： 一张动态的校园地图，所有用户选点以光点标注，密度越高的区域越亮——直觉上呈现出"哪里最需要改变"。
文案：

页面标题：所有人的校园想象
副标题：这里的每一个点，都是一个人在这座校园里的感受与期待
点击某个光点：显示对应用户的原始输入 + 生成结果缩略图
统计数据展示（增强参与感）：已有 xxx 位访客参与 · 覆盖校园 xx 个区域


## 三、整体文案语气规范
为了保持一致性，建议制定如下语气原则：
维度方向人称用"你"而非"用户"，用"我们"而非"系统"专业词汇全程回避，AI生成→"AI来理解"，参数化→不出现情感基调温暖、好奇、有点诗意，但不煽情句式短句为主，多用破折号和省略号制造呼吸感错误提示鼓励式："再多说一点点，AI会理解得更好" 而非 "输入不能为空"

## 四、关键体验设计原则
最后提炼几条贯穿始终的设计判断标准：
1. 降低第一步的门槛，而不是最后一步。 进入越容易，后续参与才越深。首屏不能有任何注册或选项。
2. 让"不专业"变成优势。 文案要反复传递：普通人的直觉感受，正是设计专业学生忽略的东西。
3. 结果必须让人感到"被理解"，而不是"被执行"。 生成结果页的AI回应文字是整个体验最重要的一段文案，需要做到个性化、有温度。
4. 集体感是留存的关键。 Gallery页的设计要让用户看到"我的点"在整个校园图景中的位置，产生"我参与了一件大事"的归属感。
