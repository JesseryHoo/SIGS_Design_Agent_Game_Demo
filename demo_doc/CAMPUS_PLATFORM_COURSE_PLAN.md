## 🎯 核心定位

### 项目名称
**"SIGS Re:Make" - 校园共创与AI改造平台**

### 一句话描述
一个让师生通过探索了解校园、通过AI畅想改造、通过投票参与决策的互动平台。

### 三层价值递进
```
Layer 1 游戏层：探索打卡了解校园现状
    ↓
Layer 2 创作层：AI辅助设计心中的理想校园
    ↓
Layer 3 共治层：提案讨论，民主参与校园建设
```

### 社会意义升华
🌱 **教育意义**：培养大学生公民意识和参与感
🤖 **技术探索**：AI+民主决策的创新模式
🏫 **校园价值**：为学校规划提供真实需求数据
🌍 **可复制性**：推广到其他高校和社区

## 🚀 极简技术方案（零成本）

### 技术栈
```yaml
前端：
  框架：Vue 3 + Vite（快速开发）
  UI：Element Plus（现成组件）
  3D：Three.js + Vue Three Fiber
  状态：Pinia
  部署：Vercel / Netlify（免费托管）

后端：
  框架：Node.js + Express / Flask（Python）
  数据库：MongoDB Atlas（免费500MB）
  认证：JWT
  部署：Render / Fly.io（免费额度）

AI功能：
  图像生成：Stability AI API / DALL-E（有免费额度）
  图像理解：GPT-4 Vision / Claude Vision
  文本生成：免费使用学校API额度

3D素材：
  获取方式：Sketchfab免费模型 / 自己用Blender做简化版
  风格：Low Poly风格（性能好，开发快）
```

### 系统架构图（极简版）

## 🎮 功能设计（MVP版本）

### 模块1：校园探索（基础层）

#### 1.1 3D校园浏览
```
功能：
- 低多边形风格的3D校园模型
- 可自由漫游查看主要建筑
- 点击建筑显示基本信息

技术实现：
- Three.js基础场景
- 5-10个主要建筑简化模型
- 鼠标控制视角和移动
```

#### 1.2 地标打卡系统
```
功能：
- 10-15个校园地标点
- 点击地标显示卡片（图片+简介）
- 知识问答（每地标1-2题）
- 打卡进度条

数据准备：
- 事先拍摄校园照片
- 每个地标准备100字介绍
- 准备20-30道问答题
```

#### 1.3 简化积分系统
```
规则：
- 打卡地标：+10分
- 答对题目：+5分
- 分数等级：铜牌/银牌/金牌探索者

展示：
- 个人中心显示积分和等级
- 简单排行榜（前10名）
```

---

### 模块2：AI校园改造（创新层⭐）

#### 2.1 智能改造建议
```
功能：
- 用户选择想改造的区域（如图书馆前广场）
- 输入改造想法："希望有更多绿植和休息区"
- AI生成改造建议和效果图

技术实现：
- 用GPT-4/Claude生成建议文本
- 调用DALL-E 3/Stable Diffusion生成效果图
- 保存为用户的改造提案
```

#### 2.2 模板式改造
```
功能：
- 提供预设改造主题
  - 🌿 绿色生态：增加植被、环保设施
  - 🎨 艺术美化：雕塑、彩绘、装置艺术
  - 💡 智能科技：智能座椅、充电站、显示屏
  - 🛋️ 社交空间：户外讨论区、休息平台
  - ♿ 无障碍改造：坡道、盲道、无障碍设施

- 用户选择主题+区域，快速生成方案
```

#### 2.3 改造方案展示
```
展示内容：
- 改造前对比图（实拍）
- AI生成的改造后效果图
- 改造说明和理由
- 预计功能和好处

交互：
- 点赞/评论
- 收藏方案
- 分享到社交媒体
```

---

### 模块3：民主参与平台（升华层⭐⭐）

#### 3.1 提案广场
```
功能：
- 展示所有用户提交的改造方案
- 分类浏览（按区域/按主题）
- 热门排序（点赞数）
- 最新排序

界面：
- 卡片式布局
- 显示改造前后对比
- 点赞数、评论数
```

#### 3.2 投票系统
```
功能：
- 每周精选3-5个优秀提案
- 用户可投票支持
- 得票高的标记为"热门提案"
- 月度冠军获得官方展示机会

意义：
- 模拟民主决策过程
- 让学生的声音被听见
- 为学校提供真实数据
```

#### 3.3 讨论区
```
功能：
- 每个提案可评论
- 支持/反对意见
- 建设性讨论
- 标记优质评论

展示：
- 评论列表
- 点赞评论
- 用户等级标识
```

## 🎨 简化3D方案

### 方案A：极简Low Poly（推荐）
```
优点：
- 开发快速（1-2周完成）
- 性能好，所有设备流畅
- 风格统一美观
- 易于维护

实现方式：
- 用基础几何体（立方体、圆柱体）
- 统一色彩风格
- 重点建筑稍微精细
- 配合文字标签
```

### 方案B：2.5D伪3D
```
优点：
- 更快速（1周完成）
- 美观可控
- 技术简单

实现方式：
- 2D平面地图
- 建筑用等距视角图
- Three.js做简单动画
- 点击交互即可
```

### 方案C：全景图漫游
```
优点：
- 最快速（3-5天完成）
- 真实感强
- 素材易得

实现方式：
- 用手机拍360°全景图
- 使用现成全景图组件
- 添加热点标记
- 简单交互
```

**建议选择方案A（Low Poly）**，平衡效果和开发时间。

### 实现代码示例

```javascript
// AI图像生成（以Stability AI为例）
async function generateDesignImage(prompt) {
  const response = await fetch('https://api.stability.ai/v1/generation/stable-diffusion-xl-1024-v1-0/text-to-image', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      text_inputs: prompt,
      cfg_scale: 7,
      height: 768,
      width: 1024,
      steps: 30,
    })
  })
  const data = await response.json()
  return data.artifacts[0].base64
}
```
